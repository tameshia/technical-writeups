# Reading Logs in Splunk: A Beginner SIEM Lab

## Objective
The goal of this lab was to begin exploring how Splunk can be used to read and interpret log data in a SIEM environment. As a beginner, I focused on reviewing log entries, identifying useful event details, and understanding how Splunk can support security monitoring.

## Tools Used
- Splunk
- Sample log data
- Web browser
- Lab environment

## Environment
This lab was completed in a beginner Splunk environment using available log data for practice. The focus of the exercise was not advanced Splunk administration, but learning how to navigate the platform and review logs in a structured way.

## Steps Performed
1. Opened Splunk and accessed the available data.
2. Navigated to the search area to begin reviewing logs.
3. Observed how log entries are displayed in Splunk search results.
4. Reviewed fields such as timestamps, hosts, sources, sourcetypes, and event messages.
5. Expanded individual events to better understand the details contained in each log entry.
6. Practiced looking for patterns and useful information that could support investigation or monitoring.

## Key Observations
- Splunk organizes log data in a way that makes large sets of events easier to search and review.
- Important details such as timestamps, hosts, and event messages help give context to activity.
- Even basic log review requires careful attention to patterns, repeated events, and unusual entries.
- Expanding individual log events makes it easier to understand what happened and what information is available for analysis.

## What I Learned
This lab helped me better understand how Splunk can be used as a SIEM tool for log review and event analysis. As a beginner, it gave me more confidence navigating the interface and recognizing the value of logs in identifying system activity and possible security concerns.

## Why It Matters
Reading logs is a foundational skill in IT and cybersecurity. Security analysts and administrators rely on logs to investigate activity, detect unusual behavior, and better understand what is happening across systems. Learning how to review logs in Splunk is an important early step toward stronger SIEM and security monitoring skills.
